Section 01

What Are We Training?

The full setup in plain English before any numbers.

Plain English — The Setup

Two AI agents learn against each other in a 20×20 grid. The Snake 🐍 earns reward by catching the bait as fast as possible. The Bait 🎯 earns reward every step it stays alive and loses heavily when caught.

Both use Deep Q-Networks (DQN) — neural networks that learn by playing thousands of games, remembering what worked and gradually building better strategies. This is co-evolutionary training: as the snake improves, the bait must adapt, which forces the snake to improve further. A true arms race. The critical requirement: both agents must keep exploring simultaneously — if either one locks in too early, the arms race ends and interesting strategies never emerge.

🐍

Snake — 4 Actions

Up, Down, Left, Right. Rewarded for catching bait quickly. Penalised every step without a catch.

🎯

Bait — 5 Actions

Up, Down, Left, Right, Stay. Rewarded every survival step. Heavily penalised when caught.

🧠

DQN Algorithm

Neural net takes the game grid as input, outputs a quality score (Q-value) for each action. Highest score wins.

♟️

Co-Evolution

Neither agent trains against a fixed opponent. Both adapt simultaneously — escalating mutual improvement.

Section 02

Epsilon (ε) — The Exploration Rate

The single number that determined the arc of both training runs.

Plain English — What Is Epsilon?

Think of learning chess. Early on you try random moves to discover what works. Later you trust what you have learned. Epsilon is the probability of making a completely random move instead of the best known one. ε=1.0 means 100% random. ε=0.05 means 95% exploitation of learned strategy.

Run 1 disaster: With decay_steps=100,000 and ~40 steps per game: 100,000÷40 = only 2,500 episodes to hit minimum epsilon. Both agents froze almost immediately — before developing any real strategy. Run 2: decay_steps=1,500,000 was intended to delay the floor until episode ~37,500. However, because games in Run 2 lasted much longer (average ~100 steps vs 40), total environment steps accumulated faster. The epsilon floor of 0.050 was actually hit at episode 21,700 — roughly 58% of the way through the run.

ε(t) = ε_end + (ε_start − ε_end) × e^(−t / decay_steps) where t = total environment steps
Run 1: floor ep ~2,300 | Run 2: floor ep 21,700 (predicted 37,500 — actual steps/ep were ~100, not ~40)

Section 03

Reward Signals — Are Agents Learning?

Reward is the fundamental learning signal. Every other metric flows from it.

Plain English — What Is Reward?

After every game each agent gets a total reward score. This is a zero-sum game — snake and bait rewards always mirror each other and sum to ~zero. Positive snake reward = snake won that game. Negative = bait survived. The trend over many episodes matters more than any single game.

Run 1: Snake reward flips permanently positive at episode ~1,000 and never looks back. The bait is stuck on −10 forever. Run 2 (eps 100–21,700): Rewards bounce — sometimes snake ahead, sometimes bait ahead. This oscillation IS the arms race. Run 2 (eps 21,700–33,500): After the epsilon floor, the bait's advantage became increasingly systematic. Note the appearance of partial rewards like −2.69 or −3.39 for the snake — these are timeout episodes where the game hit the step limit and neither agent fully won, but the bait received most of the cumulative reward.

Section 04

Win Rate — The Competition Score

The clearest single metric for co-evolutionary health — and the one that revealed the post-floor divergence first.

Plain English — What Is Win Rate?

Every 100 episodes, win rate shows what percentage of games the snake won. 50% is the ideal co-evolutionary equilibrium — both agents equally matched, both pushing each other harder. 97–100% means the arms race is over (snake won). Sub-10% means it's over the other way (bait won).

Run 2, eps 100–10,000: Gradually climbed from 5% toward 50% — healthy exploration. Run 2, eps 10,000–21,700: Oscillated in the 30–54% range — active arms race near equilibrium. Run 2, eps 21,700+: Win rate collapsed — from ~35% at the epsilon floor to below 10% by episode 29,500. The bait has locked in an evasion strategy the snake cannot overcome with only 5% exploration.

Win Rate = (Episodes snake caught bait / Window size) × 100 | Ideal equilibrium ≈ 50% ± 15%

Section 05

Loss Function — Network Learning Quality

Loss tells you whether the neural network is genuinely learning or entering a degenerate feedback loop — and Run 2 eventually shows both phases.

Plain English — What Is Loss?

The DQN predicts "if I take action X in state S, I expect reward Y." Loss is the gap between prediction and reality. The formula is Huber Loss — precise for small errors, capped for large ones. This prevents one catastrophically wrong prediction from crashing all training.

Run 2, eps 100–18,000 (healthy phase): Loss rose gradually from 0.07 to ~0.18, reflecting genuinely complex strategies emerging. Normal productive learning. Run 2, eps 18,000–21,700 (transition): Loss started climbing more steeply as epsilon dropped and the bait began winning more consistently. The snake's Q-predictions were being invalidated. Run 2, eps 22,000–33,500 (divergence): Loss surged from ~0.3 to over 1.16. The snake's network is trying to reconcile its learned pursuit strategy with a bait it can no longer catch — a degenerate reward signal loop similar to Run 1's collapse, but inverted.

Huber: L(δ) = ½δ² when |δ|≤1 | |δ|−½ when |δ|>1 where δ = Q_target − Q_predicted
Q_target = r + γ·max Q(s′,a′) | γ=0.99 (discount factor)

Section 06

Steps Per Episode — Game Duration

Game duration is a proxy for strategic sophistication — and the emergence of 200-step timeout games is the clearest signal of bait dominance.

Plain English — What Do Steps Tell Us?

Steps = moves before game ends. Short games (5–20 steps) with high win rate = snake found an immediate aggressive route. Long games (100+ steps) = bait successfully evading. Steps=200 means the game hit the maximum step limit — the bait survived the entire episode without being caught.

Run 2, eps 100–12,300: Games ranged 8–132 steps. High variance. Diverse experience. Run 2, eps 12,300–21,700: Games extended further — bait learning to survive longer (steps regularly hitting 100–167). Run 2, eps 21,700+: Steps=200 (timeout) became the dominant outcome. By episode 28,000+, the majority of games end with the bait surviving the full 200 steps. The snake cannot corner a bait that has mastered its evasion strategy.

Section 07

Co-Evolutionary Dynamics & Volatility

Measuring the arms race mathematically — including when and why it ended.

Plain English — Non-Stationarity & The Volatility Collapse

In biology, cheetahs and gazelles co-evolved over millions of years — each species improving in response to the other. In ML we replicate this by training agents simultaneously. The key mathematical challenge is non-stationarity: the optimal strategy keeps changing because whatever the snake learns, the bait adapts. You cannot simply minimise a fixed loss — the target keeps moving.

The volatility chart shows the rolling standard deviation of win rate over 5 checkpoints. High volatility = active arms race. Low volatility = one agent locked in. Run 2 maintained healthy volatility through the exploration phase (eps 100–21,700). After the epsilon floor, a second volatility collapse occurred — not from snake dominance like Run 1, but from bait dominance. The arms race ended the same way, just in the opposite direction.

F_snake(t) = f(π_snake, π_bait(t)) — snake fitness depends on bait policy which changes every episode
Volatility = σ[WinRate(t−w : t)] where w=5 checkpoints — high σ = active arms race

Section 08

The Post-Floor Divergence — What Went Wrong

Run 2 did not collapse the same way as Run 1. It revealed a second failure mode: bait dominance after epsilon drop.

🚨 A New Failure Mode

Run 1 failed because the snake stopped exploring and dominated the bait. Run 2 failed because the bait locked in its evasion strategy once epsilon dropped, and the snake — with only 5% exploration — could not overcome it. The system converged to a local equilibrium where the bait always survives to 200 steps.

This is actually a sign of successful bait training: the bait genuinely learned evasion. But the snake's Q-network is now receiving a degenerate training signal — trying to estimate reward for catching a bait it can almost never catch. This explains the loss divergence (0.07 → 1.16) and the cycling patterns in later win rate data.

⚠️ Why Did the Floor Hit at ep 21,700 Instead of ~37,500?

The prediction of ep ~37,500 assumed ~40 steps per game (from Run 1's average). Run 2's average was approximately 100 steps per game — because the bait was actively evading and games lasted much longer. More steps per episode = faster total step accumulation = faster epsilon decay.

Calculation: 1,500,000 decay_steps ÷ 100 steps/ep ≈ 15,000 episodes for the e-fold decay point. Combined with the exponential tail, ε≈0.050 was reached at ep ~21,700. This is a reminder that epsilon schedules should be set based on expected game duration, which itself changes as agents improve.

The Timeout Epidemic — Steps = 200

The emergence of 200-step timeout games is the clearest signal of bait dominance. In these episodes, the snake's reward is a partial negative (e.g. −2.69, −3.39) rather than the full −10 to −12 loss, and the bait receives a reduced positive reward. This creates a muddied training signal for both agents. The snake learns neither good pursuit nor good abandonment — it learns an ineffective middle ground. The bait, meanwhile, reinforces whatever it did to survive the full 200 steps.

🐍

Snake's Problem

With ε=0.050, 95% of moves are deterministic. The snake cannot discover new pursuit strategies to overcome the bait's locked-in evasion.

🎯

Bait's Dominance

Bait learned wall-hugging and maximising distance. Against the snake's frozen strategy, it wins ~90% of games by episode 30,000.

📊

Loss Divergence

Loss rose from 0.05 (ep 1,000) to 1.16 (ep 33,000). Q-network predictions are increasingly wrong as the reward signal becomes inconsistent.

🔧

Fix for Run 3

Higher ε floor (0.15), scheduled exploration resets, adaptive ε based on win rate, or curriculum learning with opponent pools.

Section 10

Run 3 — What To Change

Both runs revealed different failure modes. Run 3 should fix both simultaneously.

🎯

Higher ε Floor (0.15+)

ε=0.050 was too low. The snake could not explore new pursuit strategies to overcome the bait's locked evasion. A floor of 0.15 keeps 15% randomness — enough to break local optima.

🔄

Win-Rate Adaptive ε

If win rate drops below 20% for 500 episodes, temporarily boost ε to 0.3. If win rate exceeds 80%, boost bait ε. Dynamic exploration prevents single-agent lock-in.

📚

Opponent Pool

Store checkpoint policies from episodes 10K–21K (the healthy co-evolution window). Occasionally sample from this pool rather than always training against the current opponent. Prevents catastrophic forgetting.

⏱️

Step Limit Calibration

Cap episodes at 100 steps (not 200) during early training, then gradually increase. This prevents the 200-step timeout from becoming the bait's dominant strategy and keeps the reward signal clean.

The Big Picture — Both Runs Combined

Run 1 was a chess match where one player studied for 2 hours and the other for 2 minutes. Run 2 improved this dramatically — both players studied seriously. But Run 2 revealed a new problem: after the study period ended, one player (the bait) remembered the lessons better and locked in a dominant strategy the other (the snake) could no longer overcome with only 5% improvisation. Run 3 needs both longer study periods AND a mechanism to restart the arms race if either side gains permanent dominance. The goal is perpetual co-evolution, not a single winner.
